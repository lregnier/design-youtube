## Why

`backend/api` has two generated-code locations that don't follow a consistent convention: oapi-codegen output lives at `internal/api/api.gen.go`, whose package/directory name (`api`) collides with the hand-written `api/` directory at the module root that holds the OpenAPI spec and codegen config; and mockery output lives separately at `internal/mocks/`. Grouping all generated code under a single `internal/gen/` tree removes this naming collision and makes "generated vs. hand-written" obvious at a glance — a common convention in Go projects.

## What Changes

- Move `internal/api/api.gen.go` → `internal/gen/api/api.gen.go` (package name stays `api`, only the directory/import path changes); update `api/oapi-codegen.yaml`'s `output:` path accordingly.
- Move `internal/mocks/` → `internal/gen/mocks/` (package name stays `mocks`); update `.mockery.yaml`'s `dir:` accordingly.
- Update all import paths across `backend/api` that reference `internal/api` (4 files) or `internal/mocks` (7 files) to their new `internal/gen/...` locations.
- Regenerate both via `go generate ./api/...` and `mockery` to confirm the new paths/configs work end-to-end.
- No behavior changes — this is a pure code reorganization. `backend/worker` is out of scope (it has no oapi-codegen output and keeps `internal/mocks/` as-is).

### Modified Capabilities

- **`hexagonal-architecture`**: add a requirement that all generated code in `backend/api` (oapi-codegen output, mockery mocks) lives under `internal/gen/`, separate from hand-written `internal/domain/`, `internal/application/`, `internal/ports/`, and `internal/adapters/`
- **`backend-unit-tests`**: update the "Mocks are generated by mockery v2" requirement so the mocks directory is `internal/mocks/` or, for modules that group all generated code together, `internal/gen/mocks/`

## Impact

- **`backend/api/internal/api/`** → **`backend/api/internal/gen/api/`**: generated oapi-codegen package (chi server, strict-server, models)
- **`backend/api/internal/mocks/`** → **`backend/api/internal/gen/mocks/`**: generated mockery mocks
- **`backend/api/api/oapi-codegen.yaml`**: `output:` path updated
- **`backend/api/.mockery.yaml`**: `dir:` updated
- 11 Go files (4 importing `internal/api`, 7 importing `internal/mocks`, 1 overlapping) have updated import paths — no logic changes
