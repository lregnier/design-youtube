## MODIFIED Requirements

### Requirement: S3 path-style access is configurable via environment variable
The API SHALL use path-style S3 addressing when `S3_ENDPOINT_URL` is set, and virtual-hosted style (the AWS default) when it is unset. The `S3_USE_PATH_STYLE` environment variable is removed.

#### Scenario: Path-style used with custom S3 endpoint
- **WHEN** `S3_ENDPOINT_URL` is set to `"http://minio:9000"`
- **THEN** the S3 client issues requests as `http://minio:9000/bucket/key`

#### Scenario: Virtual-hosted style used in production
- **WHEN** `S3_ENDPOINT_URL` is unset
- **THEN** the S3 client uses virtual-hosted style (`https://bucket.s3.amazonaws.com/key`)

## REMOVED Requirements

### Requirement: LocalStack mode is configurable via environment variables
**Reason**: LocalStack is replaced by MinIO. The `LOCALSTACK_ENABLED` + `LOCALSTACK_ENDPOINT` variables are superseded by `S3_ENDPOINT_URL`.
**Migration**: Replace `LOCALSTACK_ENABLED=true` + `LOCALSTACK_ENDPOINT=http://localhost:4566` with `S3_ENDPOINT_URL=http://localhost:9000`.

## ADDED Requirements

### Requirement: Per-service AWS endpoint URLs are configurable via environment variables
The API SHALL read optional endpoint overrides for each AWS service: `S3_ENDPOINT_URL`, `DYNAMODB_ENDPOINT_URL`, and `SQS_ENDPOINT_URL`. When set, the corresponding AWS SDK client SHALL use that endpoint. When unset, the client SHALL use the real AWS endpoint.

#### Scenario: Custom S3 endpoint configured
- **WHEN** `S3_ENDPOINT_URL` is set to `"http://minio:9000"`
- **THEN** the S3 client sends all requests to `http://minio:9000`

#### Scenario: Custom DynamoDB endpoint configured
- **WHEN** `DYNAMODB_ENDPOINT_URL` is set to `"http://dynamodb-local:8000"`
- **THEN** the DynamoDB client sends all requests to `http://dynamodb-local:8000`

#### Scenario: Custom SQS endpoint configured
- **WHEN** `SQS_ENDPOINT_URL` is set to `"http://elasticmq:9324"`
- **THEN** the SQS client sends all requests to `http://elasticmq:9324`

#### Scenario: No overrides set in production
- **WHEN** none of `S3_ENDPOINT_URL`, `DYNAMODB_ENDPOINT_URL`, `SQS_ENDPOINT_URL` are set
- **THEN** all AWS SDK clients use real AWS endpoints

### Requirement: Presigned URL host is rewritten when a public S3 endpoint is configured
When `S3_PUBLIC_URL` is set, the API SHALL rewrite presigned upload URLs to use that host so that browser clients can reach the object store. When unset, presigned URLs are returned unchanged.

`S3_PUBLIC_URL` is distinct from `S3_ENDPOINT_URL`: `S3_ENDPOINT_URL` is the internal Docker-network address used by the SDK client (e.g. `http://minio:9000`), while `S3_PUBLIC_URL` is the host-reachable address used in URLs returned to the browser (e.g. `http://localhost:9000`).

#### Scenario: Presigned URL rewritten for local dev
- **WHEN** `S3_PUBLIC_URL` is set to `"http://localhost:9000"` and a presigned URL is generated
- **THEN** the presigned URL host is replaced with `http://localhost:9000`

#### Scenario: Presigned URL unchanged in production
- **WHEN** `S3_PUBLIC_URL` is unset
- **THEN** presigned upload URLs are returned as generated by the AWS SDK
